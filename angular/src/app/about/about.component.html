This is a Hospital Management System built using the ASP.NET Boilerplate (ABP) framework and Angular. It is designed to provide a structured, scalable and maintainable solution for managing hospital operations.

What is this Hospital Management System?

The Hospital Management System is an enterprise-style web application that helps manage doctors, patients, admissions, rooms and beds in an organized manner.

It simplifies administrative tasks such as patient registration, room allocation, bed occupancy tracking and doctor management. The system ensures proper data organization using modern software development practices.

Core Modules

The system implements several core modules:

  • Doctor Management – Manage doctor details, specialization and availability.
  • Patient Management – Register and maintain patient personal and medical information.
  • Patient Admission – Handle admission, diagnosis, doctor assignment and discharge.
  • Room Management – Maintain different room categories such as ICU, General and Private.
  • Bed Management – Monitor bed availability and occupancy status in real time.

Architecture

This project follows a clean N-Layer Architecture:

  • Domain Layer – Entities and business logic.
  • Application Layer – Application services and DTOs.
  • Infrastructure Layer – Repository and database operations.
  • Presentation Layer – Angular-based user interface.

The system also follows Domain Driven Design (DDD) principles, ensuring separation of concerns and maintainability.

Technologies Used

The application is built using modern technologies including:

  • ASP.NET Core
  • ASP.NET Boilerplate (ABP)
  • Angular
  • Entity Framework Core
  • SignalR
  • SQL Server

Purpose of the Project

The purpose of this project is to demonstrate a real-world enterprise application structure using ABP framework, including CRUD operations, DTO mapping, repository pattern, enum management and dynamic dashboards.
